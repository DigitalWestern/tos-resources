# TOS Resources

Static website for [TOSResources.com](https://tosresources.com), a free patient-friendly resource hub for Thoracic Outlet Syndrome (TOS).

The site hosts community-derived guides, downloadable files, PubMed literature search, diagnostic and treatment navigation pages, methodology notes, and external resource links. It is built to help patients find useful information while clearly separating peer-reviewed literature, clinical reference material, practical tools, and anecdotal community reports.

## Site Status

- **Live site:** [tosresources.com](https://tosresources.com)
- **Production branch:** `main`
- **Hosting:** GitHub Pages
- **Architecture:** static HTML/CSS/JavaScript
- **Build step:** none
- **Framework:** none

Pushing to `main` triggers `.github/workflows/deploy.yml`, which deploys the repository root to GitHub Pages.

## Main Pages

Primary navigation order:

```text
Home / Literature / Diagnosis / Conservative / Community / Medications / Surgery / Outreach
```

- `index.html` — homepage, Start Here routing, downloads, full-text search, red flags, site origin
- `literature.html` — PubMed literature browser using `pubmed_articles.json`
- `diagnosis.html` — diagnostic pathway, tests, imaging, specialist guidance
- `conservative.html` — conservative care, PT strategy, pacing, exercise cautions
- `community-discussion.html` — themes from the 382-post Reddit dataset
- `medications.html` — medications, injections, Botox, anticoagulation, supplements
- `surgery.html` — procedures, approaches, recovery, outcomes, surgeon references
- `outreach.html` — external resources and consolidated downloads
- `methodology.html` — sources, badge system, limitations, AI disclosure, review log
- `community.html` — redirect stub kept for old bookmarks

## Styling and Behavior

- Shared CSS lives in `css/site.css`
- Shared JavaScript lives in `js/site.js`
- Many pages still include inline CSS/JS
- `css/site.css` is loaded after inline page styles so shared rules can override equal-specificity page styles
- Production design is intentionally light-only, Times New Roman, charcoal-on-white, with hairline borders and warm off-white surfaces

Do not introduce frameworks, build tooling, dark mode, Google Fonts, or broad visual redesigns unless that is the explicit task.

## Working on the Site

Recommended orientation before meaningful changes:

1. Read `../AGENTS.md`
2. Read `../Overview.md`
3. Read `../Website Changelog.md`
4. Inspect the relevant HTML/CSS/JS files in this repo

After website changes, update `../Website Changelog.md` newest-first.

Useful checks for this static site:

- Open changed HTML locally in a browser
- Check affected links and download paths
- Check mobile layout if navigation, cards, filters, or long content changed
- Search for broken asset paths
- Run the relevant script if changing PubMed data or scripts
- Check `git status` before committing
